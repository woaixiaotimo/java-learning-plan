

# 主从复制

## 概念

1. Master(主)可以拥有多个slave(从)
2. 多个slave可以连接同一个master外，还可以链接到其他slave
3. 主从复制不会阻塞master，在同步数据时master可以继续处理client的请求
4. 提供系统的伸缩性

## 主从复制过程

1. slave与master建立链接，发送sync同步命令
2. master会开启一个后台进程，将数据库快照保存到文件中，同时master主进程会手机新的写命令并缓存
3. 后台完成保存后，就将文件发给salve
4. slave将此文件保存到硬盘上

## 主从复制配置

- 克隆master后修改slave的ip地址
- 修改配置文件：redis.conf
